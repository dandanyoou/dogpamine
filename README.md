# Dogpamine 🐶

> 알고리즘이 스스로 김 빠지는 SNS

인스타 알고리즘은 "체류시간 최대화"로 설계됐다. 그런데 사용자가 진짜 원하는 건 "재밌게 잠깐 보고 끄는 것"이다. 플랫폼 목표와 사용자 목표가 어긋나 있다. 결과는 다 안다 — 보고 나서 **"내가 뭐 봤지?"** 하는 그 허무함.

**Dogpamine 의 핵심 발상:** 알고리즘과 싸우지 말고, 알고리즘이 **스스로 김 빠지게** 만든다.

---

## 왜 기존 해결책은 안 먹히나

| 해결책 | 왜 실패하나 |
|---|---|
| 스크린타임 제한 | 알림 무시하고 계속 봄 |
| 의지력 | 작동 안 함 |
| 앱 삭제 | 3일 후 재설치 |

전부 **사용자와 싸우는** 방식. Dogpamine 은 알고리즘 자체가 김 빠지게 만들기 때문에 사용자는 통제당한다는 느낌 없이 **자연스럽게** 폰을 내려놓는다.

---

## 작동 방식 (3단계)

### ① 감지 — "지금 멍 때리며 보는 중"인 걸 안다

단순 시간 측정이 아니라 **만족도 신호 다층화**:

- **스크롤 속도 가속** — 5초/스크롤 → 1초/3스크롤 = 흥미 잃고 관성 중
- **체류 시간 감소** — 영상 평균 8초 → 2초
- **반복 노출 비율** — 본 카테고리 80% 이상 → 새 정보 획득 제로
- **시간 가중치** — 새벽 1시 + 빠른 스크롤 = 후회 확률 95%

이 4개 신호로 **포만감 지수(Satiation Score)** 를 실시간 계산.

```
satiation = 0.3 * scroll_speed
          + 0.3 * (1 - avg_dwell_time)
          + 0.2 * repeat_exposure_ratio
          + 0.2 * time_of_day_weight
```

임계점 60 돌파 시 ② 단계 작동.

### ② 개입 — 알고리즘이 일부러 재미없어진다

차단이 아니라 **자연스러운 흥미 감소**:

- **무관심 콘텐츠 섞기** — 5개 중 1개 → 3개로 비율 증가 (엑셀 튜토리얼, 농기계 리뷰 등)
- **동일 영상 재등장** — "어 이거 아까 봤는데?" 자각 유발
- **페이싱 변경** — 짧고 자극적인 릴스 → 긴 정적 콘텐츠
- **채도 페이드** — 30분에 걸쳐 5%씩 감소 (인지하기 어렵지만 매력도 ↓)

핵심: **차단하면 반발하고 다른 앱 켠다. 김이 빠지면 그냥 폰을 내려놓는다.**

### ③ 이탈 유도 — 자연스럽게 끄게

- 메타 인지 푸시 — "47개 봤어요, 그중 8개는 같은 거였어요" (사실만, 죄책감 X)
- 다음 행동 제안 — 캘린더 보고 "내일 9시 회의" 한 줄
- 대체 콘텐츠 핸드오프 — 자주 듣는 팟캐스트/책 앱으로

## 데모 시나리오 (발표 3분)

```
[0:00–0:20] 정상 사용
- 본인 취향 (강아지/음식/운동) 100% 노출
- 스크롤 속도 점점 ↑

[0:20–0:50] 첫 개입
- 포만감 지수 60 돌파
- 5개 중 1개가 엑셀 튜토리얼 / 농기계 리뷰
- 발표자: "어 왜 이런 게?"

[0:50–1:30] 김 빠짐 가속
- 무관심 콘텐츠 비율 60%로 ↑
- 동일 영상 재등장 자막
- 채도 -20%

[1:30–2:00] 메타 인지 + 이탈
- "47개 봤어요. 그중 8개는 같은 거였어요"
- 책/팟캐스트 카드 부드럽게 등장
- 발표자가 폰 내려놓음

[2:00–3:00] 기술 설명 + Q&A
```

## 시작하기 — 유저 진입 가이드

세 가지 방법으로 Dogpamine 을 경험할 수 있어요. 가장 빠른 건 ① Mock SNS, 가장 실전적인 건 ② 익스텐션.

### ① Mock SNS 데모 — 가장 빠름 (10초, 설치 없음)

브라우저로 파일만 열면 됨.

```bash
git clone https://github.com/<your-username>/dogpamine.git
cd dogpamine
open index.html        # macOS
# Windows: 더블클릭으로 index.html 열기
# Linux:   xdg-open index.html
```

조작법:
- **빠르게 스크롤** — 5분 안에 5단계 모두 진행 (`DEMO_MODE = true` 기본값)
- **`D` 키** — 디버그 패널 토글 (경과 시간 / 현재 단계 / 채도 / 본 영상 수)
- **`R` 키** — 처음부터 다시 (state 리셋)

`app.js` 11번째 줄의 `DEMO_MODE = false` 로 바꾸면 30분 사이클로 작동 (실제 사용자 경험에 더 가까움).

### ② Chrome 익스텐션 — Phase 1 (현재 시점, 작동함)

지금 작동하는 범위: **Instagram Reels (`/reels/*`)** 에서 빠르게 스크롤하면 화면 채도가 단계적으로 감소.

**설치 (개발자 모드, 1회):**

1. Chrome 주소창에 `chrome://extensions` 입력 → Enter
2. 우측 상단 **"개발자 모드"** 토글을 ON
3. **"압축해제된 확장 프로그램을 로드합니다"** 클릭
4. clone 한 폴더 안의 `extension/` 디렉토리 선택
5. 익스텐션 목록에 "Dogpamine" 카드 나타나면 성공
6. (선택) 도구바의 퍼즐 아이콘 클릭 → Dogpamine 옆 핀 아이콘 클릭으로 고정

**사용:**

- Instagram 평소처럼 로그인
- `https://www.instagram.com/reels/` 방문
- 빠르게 스크롤 → 화면 채도가 부드럽게 빠짐
- **디버깅 확인:** 페이지에서 `F12` → Console 탭 → `[Dogpamine] tick — stage=… measured=…` 5초마다 출력되면 정상 작동

**현재 한계 (Phase 1):**
- Instagram Reels 만 지원 (YouTube/TikTok 등 미지원)
- 채도 페이드만 (오버레이 / 알림 없음)
- popup UI 없음 (DevTools 콘솔로 디버깅)

### ③ Chrome 익스텐션 — v2 (구현 예정)

설계 완료, 구현 진행 중. 디자인 doc 기준 변경 사항:

- **YouTube Shorts 추가** — `youtube.com/shorts/*` 에서도 작동
- **풀스크린 오버레이** — STOP 단계 도달 시 "잠깐요. 지금 N개 봤어요, 그중 M개는 같은 거였어요" 카드 등장. "알겠어요" 클릭으로 닫기
- **popup UI** — 익스텐션 아이콘 클릭 시 현재 사이트 / 단계 / 측정 카운트 표시
- **Dev hotkey** — `Ctrl+Shift+S` 누르면 단계 강제 진행 (리허설/시연 준비용)

v2 구현 후에는 위 ② 와 동일한 방법으로 설치 (`extension/` 폴더 Load unpacked).

### 요구사항

- **Chrome 또는 Chromium 기반 브라우저** (Edge, Brave 등 가능) — Manifest V3 익스텐션이라 Firefox / Safari 미지원
- **개발자 모드 활성화 필요** — Chrome Web Store 미배포 (해커톤 일정 한계, 추후 Phase 3 에서 배포 예정)
- Mock SNS (①) 만 사용한다면 어떤 모던 브라우저든 OK

### 문제 해결

| 증상 | 원인 | 해결 |
|------|------|------|
| 채도가 안 빠짐 | 익스텐션이 URL 매칭 안 됨 | 주소창에 `/reels/` 가 정확히 포함됐는지 확인 |
| 콘솔에 `[Dogpamine]` 로그 없음 | 익스텐션 비활성 또는 로드 실패 | `chrome://extensions` 에서 토글 ON 확인, 에러 메시지 있으면 manifest.json 검토 |
| 시크릿 모드에서 안 됨 | 시크릿 모드는 별도 허용 필요 | `chrome://extensions` → Dogpamine 카드 → "세부정보" → "시크릿 모드에서 허용" ON |
| 새로고침하면 단계가 리셋됨 | 의도된 동작 | 탭 리셋 = "오늘 깨끗하게 다시" — 영구 저장은 Phase 3 에서 |
| Instagram 업데이트 후 작동 안 함 | 셀렉터 (`main video`) 마크업 변경 가능성 | Issue 등록 — content.js `videoSelector` 수정 필요 |
